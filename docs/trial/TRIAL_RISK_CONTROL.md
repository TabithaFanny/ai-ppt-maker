## 成本保护

### API 调用限制

| 操作 | 单次 Token 上限 | 说明 |
|------|-----------------|------|
| 风格提取（extract） | ~2000 tokens | 每页 StyleDNA |
| 风格提炼（distill） | ~3000 tokens | 综合 StyleKit |
| DeckPlan 生成 | ~4000 tokens | 大纲规划 |
| PPT 内容生成 | ~8000 tokens | 完整 PPT |
| AI 编辑 | ~3000 tokens | 单次编辑 |

### 建议保护措施

1. **API Key 仅服务端**：已确认 ✅
2. **单次请求超时**：120 秒 AbortController ✅
3. **文件大小限制**：50MB ✅
4. **建议添加**（生产环境）：
   - 每用户每日调用次数限制
   - 每用户每月 Token 消耗上限
   - 异常调用告警

### 试用期成本估算

假设每位试用用户：
- 上传 3 个模板 × 3 次提取 = 9 次 extract + 3 次 distill
- 生成 5 个 PPT = 5 次 DeckPlan + 5 次生成
- AI 编辑 20 次

总计约：50 次 API 调用 × 3000 tokens/次 = 150K tokens/用户

6 位用户 × 150K = 900K tokens（约 $2-5，取决于模型定价）

- 用户清除浏览器数据即清除 IndexedDB
- 无用户账号，无数据关联

## 应急预案

### 场景 1：API 费用异常

**症状**：API 调用量突然增加
**处理**：
1. 检查 MiniMax 控制台调用统计
2. 如确认异常，临时更换 API Key
3. 排查是否有自动化脚本在调用

### 场景 2：服务崩溃

**症状**：页面无法访问
**处理**：
1. 检查服务器日志
2. 重启服务 `pnpm start`
3. 如是内存问题，检查是否有大文件未清理

### 场景 3：用户报告数据丢失

**症状**：用户说生成的 PPT 不见了
**处理**：
1. 检查 IndexedDB 是否有版本历史
2. 如有，指导用户从版本历史恢复
3. 如无，记录问题并改进自动保存机制
