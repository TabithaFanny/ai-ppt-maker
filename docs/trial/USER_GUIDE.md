## 快速开始

### 1. 配置 API Key

编辑项目根目录下的 `.env.local` 文件：

```bash
ANTHROPIC_API_KEY=sk-ant-xxxxx
```

> 获取 API Key: https://console.anthropic.com/

### 2. 启动应用

```bash
pnpm install
pnpm dev
```

访问 http://localhost:3000

### 3. 五步创建 PPT

1. **上传模板** - 上传 PDF/PPT/PPTX 文件作为设计参考
2. **AI 分析** - 系统自动分析模板的设计风格
3. **输入需求** - 填写 PPT 主题、关键要点、页数等
4. **编辑内容** - 使用可视化编辑器调整内容
5. **生成预览** - 预览并导出 PPTX 文件

## 步骤详解

### Step 1: 上传模板

#### 支持格式

- PDF (.pdf)
- PowerPoint (.ppt)
- PowerPoint XML (.pptx)

#### 文件要求

- 最大文件大小: 50MB
- 建议页数: 3-30 页
- 内容: 包含设计风格的参考页面

#### 操作步骤

1. 点击上传区域或拖拽文件到虚线框内
2. 等待文件上传完成
3. 系统自动提取缩略图（前 3 页）
4. 点击「继续」进入下一步

#### 缩略图功能

- PDF 文件会自动提取前 3 页作为缩略图
- 可在上传区预览提取的页面
- 帮助确认上传的是正确的文件

---

### Step 2: AI 分析风格

系统使用 Claude Sonnet 4.6 分析模板，自动提取以下设计元素：

#### 分析内容

| 设计元素 | 说明 | 示例 |
|----------|------|------|
| 整体风格 | 商务/科技/创意/学术 | business |
| 色彩方案 | 主色、辅色、强调色、背景色、文字色 | #1a1a2e |
| 字体排版 | 标题字体、正文字体、字号 | Arial |
| 布局类型 | 单栏/双栏/全屏/居中 | double |
| 设计原则 | 设计 DNA 关键词 | ["简洁", "专业"] |

#### 操作步骤

1. 等待 AI 分析完成（实时显示进度）
2. 查看分析结果卡片
3. 可点击「重新分析」重新提取
4. 点击「继续」进入下一步

---

### Step 3: 输入需求

#### 表单字段

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| 主题 | 文本输入 | PPT 标题，1-100 字 | 是 |
| 详细描述 | 文本区域 | 补充说明，50-500 字 | 否 |
| 关键要点 | 列表添加 | 每个要点 10-100 字 | 是 |
| 页数 | 滑块 | 5-50 页 | 是 |
| 特殊要求 | 文本区域 | 其他要求 | 否 |
| 快速模板 | 单选 | 产品介绍/学术报告/商业分析 | 否 |

#### 快速模板预设

| 模板 | 适用场景 | 特点 |
|------|----------|------|
| 产品介绍 | 新产品发布、市场推广 | 强调视觉、突出卖点 |
| 学术报告 | 论文答辩、研究汇报 | 严谨结构、数据支撑 |
| 商业分析 | 市场分析、竞品研究 | 逻辑清晰、数据驱动 |

#### 关键要点添加建议

- 每个要点简洁明了
- 建议 3-7 个要点
- 避免与主题无关的内容
- 层次分明，有逻辑性

| 拖拽幻灯片 | 上下拖动调整顺序 |

| 功能 | 说明 |
|------|------|

| 类型 | 用途 | 参数 |

- 编辑后 500ms 自动保存到 IndexedDB
- 无需手动保存
- 意外关闭可恢复数据

## 数据导入导出

### 导出项目

#### 单项目导出

1. 进入项目列表页
2. 点击项目卡片上的「导出」图标
3. 自动下载 `项目名_日期.json` 文件

#### 批量导出

1. 进入项目列表页
2. 点击顶部「导出全部」按钮
3. 自动下载 `所有项目_日期.json` 文件

### 导入项目

1. 点击顶部「导入」按钮
2. 选择 JSON 文件
3. 系统自动验证并导入

## 常见问题

### 上传问题

**Q: 上传失败？**
A: 检查以下内容：
- 文件大小是否超过 50MB
- 文件格式是否为 PDF/PPT/PPTX
- 网络连接是否正常

**Q: 缩略图提取失败？**
A: PDF 缩略图提取依赖 pdf-lib 库，部分特殊编码的 PDF 可能失败，不影响主要功能。

---

### AI 分析问题

**Q: AI 分析失败？**
A: 检查以下内容：
- API Key 是否正确配置
- API Key 是否有足够额度
- 网络连接是否正常

**Q: 分析结果不准确？**
A: 可以点击「重新分析」重新提取，或手动调整风格参数。

---

### 生成问题

**Q: 生成失败？**
A: 检查以下内容：
- 需求是否填写完整（主题、关键要点必填）
- 页数是否在 5-50 范围内
- API 连接是否正常

**Q: 生成结果不符合预期？**
A: 可以在 Step 4 编辑内容，调整标题、结论、内容块。

**Q: 内容块位置错乱？**
A: 内容块使用 0-1 相对坐标，检查 position 参数是否在有效范围内。

---

### 导出问题

**Q: PPTX 导出失败？**
A: 检查浏览器控制台错误信息，确保：
- 内容块位置参数在 0-1 之间
- 浏览器支持 File API

**Q: 导出文件打不开？**
A: 确保安装了 PowerPoint 或 WPS，尝试用其他应用打开。

---

### 数据问题

**Q: 数据保存在哪里？**
A: 数据保存在浏览器的 IndexedDB 中，清除浏览器数据会丢失项目。

**Q: 如何备份数据？**
A: 使用「导出全部」功能，将所有项目导出为 JSON 文件作为备份。

**Q: 导入失败？**
A: 确保 JSON 文件格式正确，包含必需的 version 和 projects 字段。
