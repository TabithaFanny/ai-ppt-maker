# 试用问题分级标准

> Phase 13 — 2026-05-01

---

## 分级定义

### P0 — 阻断（Blocker）

**定义**：用户无法完成核心流程，功能完全不可用。

**SLA**：24 小时内修复或提供 workaround。

**示例**：
- 上传文件后页面白屏
- 生成 PPT 时服务端 500 错误
- 导出的 PPTX 文件损坏，无法打开
- 风格提取卡死不返回结果
- AI 编辑指令导致页面崩溃

**判定标准**：
- 核心流程（上传→生成→导出）中断
- 无 workaround 可用
- 影响所有用户

---

### P1 — 严重（Critical）

**定义**：核心功能可用但有明显缺陷，严重影响用户体验。

**SLA**：3 个工作日内修复。

**示例**：
- AI 编辑理解准确率 < 50%
- 导出的 PPTX 排版严重错乱
- 风格提取结果与模板明显不符
- 生成内容与主题严重偏离
- 撤销/重做功能不正常

**判定标准**：
- 功能可用但结果不可接受
- 需要多次重试才能成功
- 影响大多数用户

---

### P2 — 一般（Normal）

**定义**：功能正常但体验不佳，有改进空间。

**SLA**：下一版本修复。

**示例**：
- 生成速度较慢（> 2 分钟）
- AI 编辑偶尔不理解指令
- 导出后部分字体不匹配
- 错误提示不够友好
- 某些边界情况处理不当

**判定标准**：
- 功能基本可用
- 不影响核心流程
- 部分用户可能遇到

---

### P3 — 轻微（Minor）

**定义**：小问题或建议，不影响使用。

**SLA**：纳入需求池，排期处理。

**示例**：
- UI 文案不够准确
- 某个按钮位置不直观
- 希望支持更多场景类型
- 希望支持自定义配色
- 性能优化建议

**判定标准**：
- 不影响功能
- 体验优化类
- 新功能需求

---

## 分级流程

```
收到反馈
  ↓
是否阻断核心流程？
  ├─ 是 → P0
  └─ 否 → 是否严重影响体验？
           ├─ 是 → P1
           └─ 否 → 是否影响部分用户？
                    ├─ 是 → P2
                    └─ 否 → P3
```

## 汇总规则

### 按用户汇总

每位用户的问题列表 + 完成率 + 满意度评分。

### 按级别汇总

P0/P1/P2/P3 各多少个，分布比例。

### 按步骤汇总

哪个步骤问题最多，哪个步骤最顺畅。

### 按类型汇总

- 功能缺陷（bug）
- 体验问题（UX）
- 内容质量（AI 输出）
- 性能问题
- 功能需求
